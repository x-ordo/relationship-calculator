# BUSINESS

## 수익화 선택지 (현실적으로)

### 1) 구독이 맞는 경우
- 유료의 핵심이 **코치/상황별 문장 생성**처럼 반복 사용되는 기능일 때
- LLM 호출 비용이 발생해서 "건당"으로 받으면 거부감이 크고, **예측 가능한 ARPU**가 필요할 때

**추천 가격대(한국):** 월 3,900~9,900원 구간이 심리적 저항이 제일 낮다.

### 2) 건당 과금이 맞는 경우
- 결과물이 명확히 "1회성"이고, 구독을 싫어하는 유저가 많을 때
- 예: 공유 카드 고퀄 테마팩, PDF 리포트, 특정 템플릿(회사/연인/가족)팩

**추천:** 구독 + 인앱결제(팩) 하이브리드가 안전하다.

## 비용 산정(정확히) 방법
가격표는 모델/토큰 비용이 자주 바뀌어서 "숫자"를 고정하지 않고, 아래 식으로 계산해라.

- 1회 코치 호출 비용 = (입력 토큰 + 출력 토큰) × (토큰 단가)
- 월 원가 = (월간 호출 수/유저) × (1회 비용)
- 목표 마진 = (가격 - 원가) / 가격

**실전 팁**
- 초기에 유료는 "월 50회" 같은 상한을 걸어 원가 폭주를 막아라.
- 무료는 규칙 기반으로 충분히 재미를 줘서, 유료로 넘어갈 이유를 만든다.

## 확장 (전문가 연계)
- 심리상담/정신건강 영역은 규제/책임이 무겁다. MVP는 "자기관리 도구" 포지션.
- 다음 단계: **심리사/의사 검수 템플릿 팩**(콘텐츠 상품)부터 붙이는 게 안전하다.

## 비용/가격 모델 (초기 가정)

### 비용 드라이버
- **LLM 호출 비용**: PRO 코치 1회당 토큰 비용 + Functions 실행 비용
- **지원/운영 비용**: 문의 대응, 환불/결제, 남용(프롬프트 인젝션) 대응
- **법적/신뢰 비용**: 개인정보 최소 전송, 로그 최소화, 면책/가이드

### 구독이 맞나?
- **맞는 경우**: 사용자가 “매주/매월” 반복해서 관계를 정리하고, 코치를 자주 쓰는 패턴(회피/스트레스 루틴)
- **아닌 경우**: 1~2번만 쓰고 끝나는 일회성(이 경우 `1회권`이 더 전환률 높음)

### 권장 (MVP → 확장)
- **MVP**: FREE + `언락 코드 기반 PRO` (배포/테스트)
- **초기 과금**: `일회권(10회/30회)` + `월 구독(무제한)`을 같이 테스트
- **수익 극대화**: “월간 리포트 자동 생성 + 공유 카드 프리셋 + 코치”를 묶어 구독 가치를 만들기
